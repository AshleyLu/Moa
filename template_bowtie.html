Bowtie

DESCRIPTION

Bowtie

Targets

  • (empty) / bowtie: Run BOWTIE on an set of input files (query) vs a database index.
  • all: Recursively run through all subdirectories (use make all action=XXX to run make XXX recursively)
  • clean: Remove all job data (not the job itself). This does works only if the template has properly implemented the clean target.

PARAMETERS

(* denotes a mandatory parameter)
  • *bowtie_db: The bowtie database to use. It is allowed to define one of the bowtie database files (.[0-9].ebwt).
    Data type: file.
  • bowtie_extra_params: extra parameters to feed bowtie
    Data type: string.
  • bowtie_forward_suffix: Last part of the sequence name identifying a file with forward reads
    Data type: string, Default: "_1".
  • bowtie_input_format: Format of the input files
    Data type: set, fastq|fasta.
  • bowtie_insertsize: Expected insertsize
    Data type: float, Default: "5000".
  • bowtie_insertsize_max: Max insertsize for a paired alignment
    Data type: float, Default: "10".
  • bowtie_insertsize_min: multiplier determining the minimal acceptable value for two paired reads to be apart. If the bowtie_insertsize is 10000 and this parameter is set at 0.8, than reads that are closer together than 8000 nt are rejecte
    Data type: float, Default: "0.1".
  • bowtie_insertsize_sed: SED expression to filter the expected insertsize from the input file name
    Data type: string.
  • bowtie_output_format: Format of the output file
    Data type: set, bowtie|bam|sam.
  • bowtie_paired_ends: perform a paired end analysis. If so, the input files are expected to be of the form *_1.fastq and *_2.fastq
    Data type: set, T|F.
  • bowtie_reverse_suffix: Last part of the sequence name identifying a file with reverse reads
    Data type: string, Default: "_2".

Input parameters

  • *bowtie_input_dir: Input files for bowtie
    Data type: directory.
  • bowtie_input_extension: file extension for the files in bowtie_input_dir
    Data type: string, Default: "fastq".
  • bowtie_input_glob: glob to select a subset of files from bowtie_input_dir
    Data type: string, Default: "*".
  • bowtie_input_limit: Number of files to use, if not defined: all files
    Data type: integer.
  • bowtie_input_sort: Sort order. Choose from: u - unsorted, s - size, sr - size reverse, t - time, tr - time reverse
    Data type: set, u|s|sr|t|tr.

System parameters

  • *title: A name for this job
    Data type: string.
  • description: A longer description for this job
    Data type: string.

Advanced parameters

  • moa_postcommand: A single shell command to be executed after the Moa is finished. For more complex processing please override the moa_postprocess target in the local Makefile.
    Data type: string.
  • moa_precommand: A single command to be executed before the main operation starts. For more complicated processing, please override the moa_preprocess target in the local Makefile.
    Data type: string.
