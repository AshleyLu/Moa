Hadoop Blast

DESCRIPTION

Hadoop Blast

Targets

  • (empty) / h_blast: Runs BLAST on a hadoop cluster
  • all: Recursively run through all subdirectories (use make all action=XXX to run make XXX recursively)
  • clean: Remove all job data, not the Moa job itself, note that this must be implemented by the template.

PARAMETERS

(* denotes a mandatory parameter)
  • *h_blast_db: Location of the blast database
    Data type: file.
  • *h_blast_input_dir: location of the hadoop installation
    Data type: directory.
  • *hadoop_base: location of the hadoop installation
    Data type: directory.
  • h_blast_eval: e value cutoff
    Data type: float, Default: "1e-10".
  • h_blast_input_extension: input file extension
    Data type: string, Default: "fasta".
  • h_blast_nohits: number of hits to report
    Data type: integer, Default: "50".
  • h_blast_nothreads: threads to run blast with (note the overlap with the Make -j parameter)
    Data type: integer, Default: "1".
  • h_blast_program: blast program to use (default: blastn)
    Data type: set, blastn|blastp|blastx|tblastn|tblastx.
  • hdfs_base: htfs://SERVER:PORT for the hdfs filesystem, defaults to "hdfs://localhost:9000"
    Data type: string, Default: "hdfs://localhost:9000".

System parameters

  • *title: A name for this job
    Data type: string.

Advanced parameters

  • moa_postcommand: A single shell command to be executed after the Moa is finished. For more complex processing please override the moa_postprocess target in the local Makefile.
    Data type: string.
  • moa_precommand: A single command to be executed before the main operation starts. For more complicated processing, please override the moa_preprocess target in the local Makefile.
    Data type: string.
