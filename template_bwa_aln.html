BWA align

DESCRIPTION

BWA align

Targets

  • (empty) / bwa_aln: Use bwa to aling a set of fastq reads against a db
  • all: Recursively run through all subdirectories (use make all action=XXX to run make XXX recursively)
  • clean: Remove all job data, not the Moa job itself, note that this must be implemented by the template.

PARAMETERS

(* denotes a mandatory parameter)
  • *bwa_aln_db: bwa database to align against
    Data type: file.
  • bwa_aln_seed_len: Seed length
    Data type: integer.

Input parameters

  • *bwa_aln_input_dir: Fastq input files
    Data type: directory.
  • bwa_aln_input_extension: file extension for the files in bwa_aln_input_dir
    Data type: string, Default: "fq".
  • bwa_aln_input_glob: glob to select a subset of files from bwa_aln_input_dir
    Data type: string, Default: "*".
  • bwa_aln_input_limit: Number of files to use, if not defined: all files
    Data type: integer.
  • bwa_aln_input_sort: Sort order. Choose from: u - unsorted, s - size, sr - size reverse, t - time, tr - time reverse
    Data type: set, u|s|sr|t|tr.

System parameters

  • *title: A name for this job
    Data type: string.

Advanced parameters

  • moa_postcommand: A single shell command to be executed after the Moa is finished. For more complex processing please override the moa_postprocess target in the local Makefile.
    Data type: string.
  • moa_precommand: A single command to be executed before the main operation starts. For more complicated processing, please override the moa_preprocess target in the local Makefile.
    Data type: string.
