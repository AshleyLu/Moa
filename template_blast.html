Basic Local Alignment Tool

DESCRIPTION

Basic Local Alignment Tool

Targets

  • (empty) / blast: Wraps BLAST [[Alt90]], probably the most popular similarity search tool in bioinformatics.
  • all: Recursively run through all subdirectories (use make all action=XXX to run make XXX recursively)
  • clean: Remove all job data, not the Moa job itself, note that this must be implemented by the template.
  • blast_report: Generate a text BLAST report.

PARAMETERS

(* denotes a mandatory parameter)
  • *blast_db: Location of the blast database. You can either define the blast db parameter as used by blast, or any of the blast database files, in which case the extension will be removed before use
    Data type: file.
  • blast_eval: e value cutoff
    Data type: float, Default: "1e-10".
  • blast_gff_blasthit: (T,**F**) - export an extra blasthit feature to the created gff, grouping all hsp (match) features.
    Data type: set, T|F.
  • blast_gff_source: source field to use in the gff
    Data type: string, Default: "BLAST".
  • blast_nohits: number of hits to report
    Data type: integer, Default: "50".
  • blast_nothreads: threads to run blast with (note the overlap with the Make -j parameter)
    Data type: integer, Default: "2".
  • blast_program: blast program to use (default: blastn)
    Data type: set, blastx|blastn|blastp|tblastn|tblastx.

Input parameters

  • *blast_input_dir: Directory with the BLAST input files
    Data type: directory.
  • blast_input_extension: file extension for the files in blast_input_dir
    Data type: string, Default: "fasta".
  • blast_input_glob: glob to select a subset of files from blast_input_dir
    Data type: string, Default: "*".
  • blast_input_limit: Number of files to use, if not defined: all files
    Data type: integer.
  • blast_input_sort: Sort order. Choose from: u - unsorted, s - size, sr - size reverse, t - time, tr - time reverse
    Data type: set, u|s|sr|t|tr.

System parameters

  • *title: A name for this job
    Data type: string.

Advanced parameters

  • moa_postcommand: A single shell command to be executed after the Moa is finished. For more complex processing please override the moa_postprocess target in the local Makefile.
    Data type: string.
  • moa_precommand: A single command to be executed before the main operation starts. For more complicated processing, please override the moa_preprocess target in the local Makefile.
    Data type: string.
