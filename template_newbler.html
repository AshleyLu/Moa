Newbler

DESCRIPTION

Newbler

Targets

  • (empty) / newbler: Run a simple, out of the box, newbler assembly. As an extra feature, this template automatically creates uniquely named links to the two main output fasta files (454AllContigs.fna, 454LargeContigs.fna). This is convenient for subsequence gather steps. The links are named after the directory.
  • all: Recursively run through all subdirectories (use make all action=XXX to run make XXX recursively)
  • clean: Remove all job data (not the job itself). This does works only if the template has properly implemented the clean target.

PARAMETERS

(* denotes a mandatory parameter)
  • newbler_largecontig_cutoff: min length of a contig in 454LargeContigs.fna
    Data type: integer.
  • newbler_library_name: A library identifier for this assembly. This is used to create an extra fasta file, named using this variable, that contain the generated contigs with their ids prepended with the library id.
    Data type: string, Default: "tmp7aa936".
  • newbler_mid_configuration: Mid configuration file to use
    Data type: file.
  • newbler_mids: mids to use for this assembly
    Data type: string.
  • newbler_min_identity: Minimal overalp identity used during assembly
    Data type: integer.

Input parameters

  • *newbler_input_dir: input SFF files
    Data type: directory.
  • newbler_input_extension: file extension for the files in newbler_input_dir
    Data type: string, Default: "sff".
  • newbler_input_glob: glob to select a subset of files from newbler_input_dir
    Data type: string, Default: "*".
  • newbler_input_limit: Number of files to use, if not defined: all files
    Data type: integer.
  • newbler_input_sort: Sort order. Choose from: u - unsorted, s - size, sr - size reverse, t - time, tr - time reverse
    Data type: set, u|s|sr|t|tr.

System parameters

  • *title: A name for this job
    Data type: string.
  • description: A longer description for this job
    Data type: string.

Advanced parameters

  • moa_postcommand: A single shell command to be executed after the Moa is finished. For more complex processing please override the moa_postprocess target in the local Makefile.
    Data type: string.
  • moa_precommand: A single command to be executed before the main operation starts. For more complicated processing, please override the moa_preprocess target in the local Makefile.
    Data type: string.
